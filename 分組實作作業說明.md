## 主題 A：旅遊顧問 MCP Server

**情境**：旅行前的一站式資訊查詢

**Tools（選 3-5 個）**

| Tool             | API                                                  | 說明             |
| ---------------- | ---------------------------------------------------- | ---------------- |
| `get_weather`  | `https://wttr.in/{city}?format=j1`                 | 查詢目的地天氣   |
| `get_fun_fact` | `https://uselessfacts.jsph.pl/api/v2/facts/random` | 旅途趣味冷知識   |
| `get_advice`   | `https://api.adviceslip.com/advice`                | 旅行前的人生建議 |
| `get_trivia`   | `https://opentdb.com/api.php?amount=1`             | 旅途知識問答     |
| `get_activity` | `https://bored-api.appbrewery.com/random`          | 推薦活動         |
| `web_search`   | `duckduckgo-search`                                | 搜尋景點、美食   |

**Resource（必須實作 1 個）**

```python
@mcp.resource("info://travel-tips")
def get_travel_tips() -> str:
    """旅行必帶物品與注意事項清單"""
    return (
        "旅行必帶物品：\n"
        "- 護照 / 身分證\n"
        "- 當地貨幣或信用卡\n"
        "- 備用藥品\n"
        "- 充電器與轉接頭\n\n"
        "出發前注意：\n"
        "- 確認當地天氣，準備適當衣物\n"
        "- 查詢當地緊急電話\n"
        "- 備份重要文件"
    )
```

**Prompt（必須實作 1 個）**

```python
@mcp.prompt()
def plan_trip(city: str) -> str:
    """產生旅遊行前簡報的提示詞"""
    return (
        f"我要去 {city} 旅行，請幫我準備一份完整的行前簡報：\n"
        f"1. 查詢 {city} 的天氣，判斷需要帶什麼衣物\n"
        f"2. 給我一則旅遊相關的冷知識或趣味資訊\n"
        f"3. 給我一則旅行前的人生建議\n"
        f"4. 推薦 2-3 個在 {city} 可以做的活動\n"
        f"請用繁體中文，語氣活潑。"
    )
```

## 分組實作流程

**Step 1：組長建立 Repo**

1. Fork [t11003/MCP_repo](https://github.com/t11003/MCP_repo) 到自己帳號，改名為 `mcp-第X組`
2. Settings → Collaborators → 邀請所有組員
3. Settings → Branches → 設定 Branch Protection：main 需要 **1 人 approve** 才能 merge

各組員 clone 並建立虛擬環境：

```bash
git clone https://github.com/組長帳號/mcp-第X組.git
cd mcp-第X組
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

---

**Step 2：各自開發 Tool**

每位組員在自己的 feature branch 開發：

```bash
git checkout -b feature/weather-tool

# 1. 在 tools/ 建立你的 Tool 檔案
# 2. 在 server.py 裡 import 並用 @mcp.tool() 註冊
# 3. 啟動 Server 測試
python server.py

git add tools/weather_tool.py server.py
git commit -m "feat: 新增天氣查詢 MCP Tool"
git push origin feature/weather-tool
```

完成後到 GitHub 開 PR，指定一位組員 review。

---

**Step 3：Code Review + Merge**

- Reviewer：在 **Files changed** 留至少 1 條 comment（重點看 docstring 是否清楚）
- 沒問題後 **Approve**，由 PR 作者 **Merge**
- 每次 merge 後，其他人先 `git pull origin main` 再繼續

---

**Step 4：加入 Resource 與 Prompt**

組長或指定組員負責，在 `server.py` 加入 Resource 和 Prompt：

```bash
git checkout -b feature/resource-prompt
# 參考上方各主題的範例，在 server.py 加入 @mcp.resource() 和 @mcp.prompt()
git add server.py
git commit -m "feat: 新增 Resource 與 Prompt"
git push origin feature/resource-prompt
# 開 PR → Review → Merge
```

**測試**：啟動 Server 後，在 Inspector 確認三個分頁都有內容：

- **Tools** 分頁：看到所有 Tool
- **Resources** 分頁：看到 Resource，點擊可讀取內容
- **Prompts** 分頁：看到 Prompt，可填入參數預覽

---

**Step 5：整合測試**

```bash
git pull origin main
python server.py   # 終端機 1
```

用 Inspector（SSE 模式，URL: `http://localhost:8000/sse`）確認三種能力都正常。

---

**Step 6：用 AI 寫 Agent**

組長或指定組員負責：

1. 打開 `agent.py`，把裡面的設計指令貼給 AI（Copilot / Antigravity）
2. 用 AI 產生的程式碼取代 `agent.py` 內容
3. 建立 `.env`，填入 `GEMINI_API_KEY`
4. 執行 `python agent.py`（終端機 2），測試對話與 `/use` 指令

```bash
git checkout -b feature/agent
python agent.py   # 測試：一般對話 + /prompts + /use plan_trip Taipei
git add agent.py
git commit -m "feat: 新增 Gemini Agent（MCP Client）"
git push origin feature/agent
# 開 PR → Review → Merge
```

---

**Step 7：填 README + 展示**

```bash
git checkout -b docs/readme
# 填寫 README.md 各欄位 + CONTRIBUTORS.md
git add README.md CONTRIBUTORS.md
git commit -m "docs: 填寫分組心得"
git push origin docs/readme
# 開 PR → merge → 完成
```

- 展示：Inspector 三個分頁截圖 + Agent 對話截圖（包含使用 `/use` Prompt 的結果）
- **繳交：GitHub repo 連結（能看到所有 PR 紀錄）**
